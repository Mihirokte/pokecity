# Memory - PokéCity (~/mihir/pokecity)

## Project
Gamified productivity suite — Pokémon city UI, React 19 + TS + Vite 7, Google Sheets as backend, Zustand state, GitHub Pages deploy.

## Architecture Quick-Ref
- **Main flow**: App → LandingPage (OAuth) → LoadingScreen → CityView → CatanCityScene (3D Catan hex board)
- **3D scene**: Hexagonal board with resident Pokémon sprites on home tiles (Three.js + react-three-fiber)
- 3 Zustand stores: authStore, cityStore, uiStore
- sheetsService.ts is the single API layer (all CRUD)
- 9 Google Sheets (core): Meta, Houses, Residents, + 6 module data sheets
- 6 modules: calendar, tasks, notes, travel, gym, shopping
- No router — view determined by auth state + selected agent
- Optimistic updates — local state first, async Sheets sync
- Cross-device sync via manual spreadsheet ID sharing

## Dev Commands
- `npm run dev` — Vite dev server on :5173
- `npm run build` — TS check + production build
- `npm run lint` — ESLint
- Needs `VITE_GOOGLE_CLIENT_ID` in `.env`

## Workflow Preferences
- After every change, commit and push to the repo immediately
- Push to main triggers GitHub Pages deploy automatically (via `.github/workflows/deploy.yml`), so pushing = deploying
- All bash commands pre-approved globally (added `Bash(*)` to `.claude/settings.local.json`) — I proceed confidently with bash; only safety prompts for genuinely destructive operations

## Key Gotchas
- `update()` in sheetsService does a full read to find row index before writing
- Trip legs and packing lists stored as JSON strings in cells
- Scopes versioned (v6) — changing scopes forces all users to re-login
- Three.js is in dependencies but was removed from UI (only sprites now)
- Base path is `/pokecity/` — affects asset paths and OAuth redirect URIs
- No conflict resolution on multi-device — last write wins

## Git History Patterns
- Iterative UI refinements (3D scene → removed → gradient + sprites)
- Cross-device sync evolved: Drive API → manual linking
- Mobile viewport fixes were recurring issues
